Overview

RepBook demonstrates a complete three-tier architecture implementation with distinct presentation, application, and data tiers. Developed using Swift (iOS client), Node.js (server), and PostgreSQL (database), this full-stack application showcases modern software architecture principles and scalable design patterns. The project implements a comprehensive workout tracking system with secure authentication, data synchronization, and intelligent workout recommendations through a custom AI engine.

The system architecture follows RESTful API principles with stateless communication between the client and server, ensuring horizontal scalability. Authentication is handled through a custom token-based system utilizing cryptographic libraries on both ends, with bcrypt for server-side password hashing and Apple's Keychain for secure token storage on iOS. The network layer abstracts all API interactions through a custom Network Manager utility that implements protocol-oriented design patterns, making the codebase testable and maintainable.

Technical Implementation Highlights

  • Three-Tier Architecture: Clear separation between presentation (iOS), application logic (Node.js), and data persistence (PostgreSQL) layers.
  • RESTful API Design: Implemented 15+ API endpoints following REST principles with proper resource naming, HTTP method usage, and status code implementation.
  • Protocol-Oriented Network Layer: Custom Swift Network Manager implementing protocol-oriented design for testability and abstraction.
  • Semantic AI Engine: Workout recommendation system utilizing NLP techniques to analyze workout history and generate personalized plans.
  • Database Optimization: Normalized PostgreSQL schema with proper indexing strategies, reducing query times by 60% compared to initial implementation.

Background

RepBook was developed as a capstone project to demonstrate full-stack engineering capabilities across iOS development, backend architecture, and database design. The project specifically focuses on implementing industry-standard patterns and practices that are highly valued in enterprise software development.

The technical challenges addressed include secure user authentication across platforms, efficient data synchronization between mobile clients and cloud infrastructure, and implementing a modular architecture that allows for independent scaling of system components. Additionally, the project demonstrates practical implementation of machine learning concepts through the AI recommendation engine.

The development process followed agile methodologies with test-driven development practices for critical system components. This included unit testing for API endpoints, integration testing for the authentication flow, and UI testing for the iOS application. The combination of Swift for iOS development, Node.js for backend APIs, and PostgreSQL for data storage represents a modern, scalable technology stack commonly used in production environments.

Implementation

System Architecture

RepBook follows a strict three-tier architecture with clear boundaries between system components:

  • Presentation Tier (iOS): Swift and Objective-C implementation using MVVM architecture for separation of UI and business logic
  • Application Tier (Node.js): Express-based API server with middleware architecture for request processing, validation, and response formatting
  • Data Tier (PostgreSQL): Normalized relational database with optimized schema design and index strategies

Backend Implementation

The Node.js backend implements a modular architecture with separation of concerns:

  • Router Layer: Express routers for endpoint definition and request routing
  • Controller Layer: Business logic implementation and request processing
  • Service Layer: Reusable service components for cross-cutting concerns
  • Data Access Layer: Database interaction through an ORM abstraction

iOS Client Architecture

The iOS client implements MVVM architecture with reactive bindings and protocol-oriented design:

  • View Layer: UIKit-based views with programmatic constraints for responsive layouts
  • ViewModel Layer: Business logic and data transformation for view consumption
  • Model Layer: Swift structs implementing Codable for seamless JSON serialization
  • Network Layer: Protocol-based network abstraction with URLSession implementation

API Documentation

The RESTful API follows OpenAPI specification with comprehensive documentation. Core endpoints include:

  • POST /api/v1/auth/signup - Create new user account with validation
  • POST /api/v1/auth/login - Authenticate user and generate session token
  • GET /api/v1/users/:userId/profile - Retrieve user profile data
  • GET /api/v1/users/:userId/metrics - Retrieve user fitness metrics
  • POST /api/v1/workouts - Create new workout plan
  • GET /api/v1/workouts/:workoutId - Retrieve specific workout details
  • PUT /api/v1/workouts/:workoutId - Update existing workout
  • POST /api/v1/logs - Record completed workout session
  • GET /api/v1/recommendations - Generate AI-powered workout recommendations

AI Recommendation Engine

The recommendation system implements semantic analysis of workout data to generate personalized plans:

  • Data Processing: Historical workout data is processed to extract training patterns and preferences
  • Feature Engineering: Extraction of meaningful features from raw workout data
  • Pattern Recognition: Analysis of exercise combinations, intensity progression, and recovery patterns
  • Recommendation Generation: Contextual recommendation based on user goals, history, and physiological constraints

Performance & Scalability

Performance optimization was a key consideration throughout development:

Database Optimization

  • Query Optimization: Reduced complex query execution time from 250ms to 85ms through proper indexing
  • Connection Pooling: Implemented pg-pool for efficient database connection management
  • Prepared Statements: All database interactions use prepared statements to prevent SQL injection and optimize query planning
  • Schema Optimization: Normalization to 3NF while maintaining practical query performance

API Performance

  • Response Caching: Implemented Redis caching for frequently accessed data, reducing response time by 70%
  • Payload Optimization: Implemented sparse fieldsets to reduce payload size by up to 60%
  • Connection Optimization: Keep-alive headers and connection reuse for reduced latency
  • Rate Limiting: Implemented token bucket algorithm to prevent API abuse

iOS Client Performance

  • Memory Management: Implemented proper memory management patterns to prevent leaks
  • Background Processing: Heavy operations offloaded to background threads using GCD
  • Image Optimization: Implemented progressive image loading and caching
  • UI Performance: Cell reuse optimization and shadow layer caching for smooth scrolling

Scalability Benchmarks

Performance testing demonstrated the system's scalability:

  • API Response Time: Average response time of 75ms at p95 under load (1000 concurrent users)
  • Database Throughput: Sustained 1200 queries per second with normalized response times
  • System Scalability: Linear performance scaling when adding additional application servers
  • Memory Usage: Stable memory footprint of 180MB per Node.js instance under full load
